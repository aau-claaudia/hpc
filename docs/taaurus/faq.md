# Frequently Asked Questions

## Access and eligibility

??? info "Are students allowed on TAAURUS?"

    At present, students are not granted direct access to TAAURUS.

    TAAURUS has a high security classification, and AAU has limited options for enforcing sanctions toward students if sensitive data is leaked.

    One possible workaround is to employ students under formal terms and conditions (for example confidentiality obligations). You can read about student recruitment options here:
    [Recruitment of students - AAU HR](https://www.en.hr.aau.dk/information-and-guidance/recruitment-of-students)

??? info "Can I set roles and rights when adding members to a TAAURUS project?"

    In the application form, you assign each person a specific project role.

    The available role options are:

    - Principal Investigators (PI)
    - Project Administrators
    - Project Member (Read/Write access)
    - Project Member (Read-Only access)

??? info "Can access privileges be updated after access is granted?"
    Yes. It will be possible to modify an active project via a form on serviceportal.aau.dk.

    This form supports (PI only):

    - role and rights changes
    - adding or removing project group members

## Project lifecycle and administration

??? info "Can I extend a running project?"
    Yes. It will be possible to request an extension via a form on serviceportal.aau.dk.

??? info "Can I add or remove members once a project is running?"
    Yes. It will be possible to add or remove members via a form on serviceportal.aau.dk.

    This form supports (PI only):

    - adding or removing project group members
    - role and rights changes

??? info "Can I request software that is not currently available on TAAURUS?"
    Yes. It will be possible to request additional software via a form on serviceportal.aau.dk.

## Data and compliance

??? info "Can TAAURUS be used only for data storage?"
    TAAURUS can store both existing data and data generated during a project.

    However, TAAURUS is not intended to be used only as archival storage. You must also have a relevant processing/compute purpose for using the platform.

??? info "Is a Data Management Plan (DMP) required, and how do we get started?"
    Yes. A DMP is required before you submit Part 2 in the [application process](/taaurus/how-to-access/).

    To get started with DMP work:

    - [Read AAU guidance on data management planning](https://www.researcher.aau.dk/guides/research-data-and-software/data-management/data-management-planning#8694afcb-4ffe-4ab2-810c-caaa472a7e38)
    - [Request support from CLAAUDIA](https://serviceportal.aau.dk/serviceportal?id=sc_cat_item&sys_id=66738c41c3c34610f0f3041ad001310d)

??? info "Do we need ethical approval and GDPR registration to get access?"
    Yes. Both are required.

    These items do not need to be fully approved before submitting Part 1, but they must be approved before submitting Part 2 in the [application process](/taaurus/how-to-access/).

    - [Read more about GDPR registration](https://aaudk.sharepoint.com/sites/persondata-ressourcer/SitePages/Anmeldelse%20og%20registreringer.aspx)
    - [Read more about ethical approval](https://forms-intern.aau.dk/dialogue/AAU084/Ansgning_om_forskningsetisk_godkendelse)

## Project structure

??? info "Can we run one large TAAURUS project with multiple sub-projects?"
    In general, no.

    If each sub-project has its own GDPR registration and ethical approval, each sub-project should run as its own separate TAAURUS project.

    Reasons:

    - projects run in isolated environments
    - separate read/write permissions cannot be enforced between sub-folders inside one project
    - each project has one primary PI with full administrative rights (including role management, import/export permissions, and project deletion)

    Because of this, it is not recommended to place multiple independent projects under one shared project owner unless that PI is directly affiliated with all included work.

## Capacity and pricing

??? info "What is the price of TAAURUS?"
    At the moment, there is no internal chargeback for standard out-of-the-box TAAURUS usage. The platform is centrally funded.

    Current CPU/GPU resources are shared across all TAAURUS projects.

??? info "Is it possible to use external/internal funding to buy additional compute for TAAURUS?"
    Yes. Additional compute capacity can be purchased and added to the shared CPU/GPU cluster.

    Your project receives prioritized access to the added capacity. When your project is not using it, the capacity can be used by other TAAURUS users.

    If you are interested, please submit a service request:
    [Service request form](https://aaudev.service-now.com/serviceportal?id=sc_cat_item&sys_id=a05e2fb4c3434610f0f3041ad001310e)
